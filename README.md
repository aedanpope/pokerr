# Pokerr

A web app for viewing poker ranges exported from [Range Manager](https://range-manager.com/).

## Project structure

```
/
├── rm_to_pokerr.py       # converts a .rm export → Pokerr JSON format
├── web/
│   ├── index.html        # app shell
│   ├── style.css         # styles
│   └── app.js            # range tree + hand builder logic
├── data/
│   └── prod.json         # active range data (generated by rm_to_pokerr.py)
└── rm_to_opt_md.py       # alternate converter → OPT-compatible Markdown/HTML
```

## Converting a Range Manager export

```bash
python rm_to_pokerr.py <input.rm> [output.json] [--title "My Title"]
```

Output defaults to `data/<stem>.json`. Examples:

```bash
# produce data/prod.json with a nice title
python rm_to_pokerr.py myRangeManagerProject.rm data/prod.json --title "GTO Ranges"

# output to a custom path
python rm_to_pokerr.py myRangeManagerProject.rm data/6max.json --title "6-Max Cash"
```

## Running locally

```bash
python -m http.server 8080
```

Then open:

```
http://localhost:8080/web/
```

By default the app loads `data/prod.json`. To load a different file:

```
http://localhost:8080/web/?data=../data/6max.json
```

## Production (GitHub Pages)

The `web/` directory is served as the site root. `data/prod.json` is fetched
relative to the repo root. No build step required.

## Pokerr JSON format

The intermediate format produced by `rm_to_pokerr.py`:

```json
{
  "format": "pokerr",
  "version": 1,
  "title": "GTO Ranges",
  "rangeMeta": {
    "<id>": { "name": "Raise Value" }
  },
  "tree": [
    {
      "type": "folder",
      "name": "RFI",
      "children": [
        {
          "type": "category",
          "name": "UTG",
          "children": [
            {
              "type": "tab",
              "name": "UTG",
              "ranges": [{ "id": "<id>", "hands": ["AA", "KK", "..."] }]
            }
          ]
        }
      ]
    }
  ]
}
```

Action colors (raise, call, fold, bluff) are determined by CSS classes in
`web/style.css` based on the `name` field in `rangeMeta` — no color values
are stored in the JSON.

## Range Manager hierarchy

```
Project (.rm / .json)
└── Folder          (e.g. "Facing RFI")
    └── Category    (e.g. "BTN")
        └── Tab     (e.g. "vs CO")
            └── Range lists per action (Raise Value, Call, Raise Bluff, Fold)
```

Both `.rm` and `.json` exports from Range Manager use identical JSON
formatting — either works as input to `rm_to_pokerr.py`.

## Open Poker Tools format

Range strings are rendered as comma-separated hand notation:

```
AA,KK,QQ,JJ,TT,99,AKs,AQs,AKo
```

Click any range string in the app to copy it to clipboard.
